Laboratory: Intro to Git

Summary: In this laboratory, you will be practicing using git. You can either use the command line (helpful commands above) to manage your git repositories or we can do it straight from Eclipse. Eclipse has a plugin, EGit that makes it easier to interact with GitHub straight from Eclipse instead of using the command line which can sometimes get messy and confusing. If you do though opt to go the command line route in the future, here's a list of commands you might find useful:

git config --global user.name “Your Name” git config --global user.email your_email@example.com git config --global core.editor editor git help git clone git add git commit git pull git push

Exercises

Setting Up

Exercise 0: Preparation

Make sure you’ve read the Intro to Git reading.

Exercise 1: Make an Account

Make a github account at https://github.com/

Exercise 2: Create a Workspace

Create a folder on your computer where you’ll store all your course files that you want to upload to git. Name it “Workspace” for example.

Exercise 3: Putting your files in one place

Go into Eclipse Window > Preferences. This opens a new window, go to Team, click the open button to its left, and click Git. Change the default repository folder to the folder you just created. Click Apply.

Exercise 4: Configuring Git

Now click the open button for Git and go to Configurations and make sure that under user it says email youremail@grinnell.edu name your_name

If it doesn’t you should add new entries for both those values. Click on Add Entry and put user.name as the Key and your username on GitHub as the Value. Click on Add Entry again and put user.email as the Key and the email you used for GitHub as the Value.

Exercise 5: Connecting Eclipse to GitHub

Stay in Preferences and open General, open Network Connections and go to SSH2 Click the Key Management then Generate RSA Key. Copy the key to your clipboard. Click Save Private Key to save the key and click Apply. Exit the preferences.

Creating Your First Repository

Exercise 6: Connecting Eclipse to GitHub

Go to the your Account Settings on the GitHub website. Under SSH Keys click Add SSH key. Give it a descriptive name (EGit), and paste the key you just copied from Git.

Exercise 7: Creating a New Repo

Create a new repository. You do this by clicking the Create a new repo button located next to you username on top of the GitHub website. Make it public and check Initialize this repository with a README.

Exercise 8: Grabbing Your Repo’s Link

Copy the SSH URI located on the top next to the HTTP | SSH | Git Read-Only tabs. Make sure you’ve selected the SSH tab.

Example: git@github.com:username/RepoName.git

Exercise 9: Getting the Repo to Show up in Eclipse

Go into Eclipse > File > Import ang click Git then go to Projects from Git then URI and paste the URI you just copied from GitHub, make sure the Host is github.com and the Repository path is username/RepoName.git. Under Authentication the User should be set to “git” and not your actual username and password. Make sure the protocol is set to blank. Click Next.

Exercise 10: Getting the Repo to Show up in Eclipse

Make sure that master is checked and click Next. Make sure the Directory the file’s going into is the folder you created at the beginning. Leave the Initial branch as master and the Remote name as origin. Click Next.

Exercise 11: Syncing it with the Package Explorer

You’ve added the repository to Eclipse. Now we want to make an accompanying project in the Package Explorer. You should be presented with a wizard to create a project in your Package Explorer. Choose Use the New Project Wizard. To view all your Git repositories go to Window > Show View > Other. Open Git and click on Git Repositories. You should now be able to view your newly added repository.

Exercise 12: Syncing it with the Package Explorer

Select the new project to be a Java Project. Open Java, choose Java Project and click Next.

Exercise 13: Naming the Project

Give it an appropriate name (you should probably name it the same as you named your repo, just to avoid confusion in the future).

Exercise 14: Finishing it up

Untick Use default location and click Browse. Now navigate to your new repository. Click Finish.

Exercise 15: Marvel at how Long It Took to Get That Set up

The new project should now appear in the Package Explorer. It is connected to the git repository and you can add new files, delete them, and modify them as you please.

Exercise 16: Getting Changes to Appear on GitHub

We need to commit these changes to the repo. Right click on your new project in the Package Explorer. Go to Team > Commit Select all the files that have been changed and type in a Commit message (for example: setting up) and click Commit and Push.

Exercise 17: FIN!

You’re done. You should now be able to view your modified project in your GitHub repo.

Getting an Already Existing Repository

Getting an Already Existing Repository

When you want to work in a group or contribute to someone else’s repository, you might not be the one to create it. There are two ways of getting an already existing repository. You can either:

  • Fork it from someone else’s repository and follow the steps above
  • Import their project directly

Forking a Repository

Exercise 18: Intro to Forking

To fork someone else’s repository start by going to https://github.com

Exercise 19: More Forking

On the website navigate to the repository that you want to work on.

Exercise 20: Still More Forking

In the upper right corner there’s a button Fork. Click it and fork it to your account. As GitHub processes the Fork request, there will be a picture of a book being scanned with a fork stuck in it.

Exercise 21: Your First Fork

In the upper right corner there’s a button Fork. Click it and fork it to your account. As GitHub processes the Fork request, there will be a picture of a book being scanned with a fork stuck in it.

Exercise 22: Getting it on Eclipse

Follow step 9 through 11 of section Creating Your First Repository.

Exercise 23: Getting it on Eclipse

For step 12, choose Import from Existing Project instead of choosing Use the New Project Wizard.

Exercise 24: Getting it on Eclipse

Follow steps 13 through 18 to get it in Eclipse and start contributing.

Importing Directly

Importing Directly

When you import directly from someone else’s repository you might not be able to push to that repository unless they’ve given you special permission. In that case you can either send the person a request for them to accept your push or just fork the repository and work on it yourself (giving them proper credit, of course).

Exercise 26: Importing directly

You need to go to the repository you want to grab on GitHub and copy the SSH URI link.

Exercise 27: Getting it in Eclipse

In Eclipse you need to go to File > Import and open Git. Choose Projects from Git and click Next. Choose URI and click Next. Enter the URI you just copied and click Next and Next again. Make sure the it goes into the folder (directory) that you’ve designated for the class and click Next. Choose Import Existing Project.

Exercise 28: Showing the Project in the Package Explorer

You should be able to see the repository in the Git Repositories view as well as the Package Explorer.

Exercise 29: Changing the Project

You can edit it freely from the Package Explorer.

Exercise 30: Pushing the Changes

After you’re done you need to commit the changes: right click on the project, go to Team and select Commit. Type in your commit message and choose Commit and Push.
